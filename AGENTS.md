# 仓库智能体指令

## 适用范围

本文件中的指令适用于整个仓库。

## 构建与验证

- 严禁在本地安装 xmake，包括但不限于通过 Scoop、Chocolatey、winget、安装程序、下载压缩包或其他包管理器安装。
- 严禁为本仓库运行任何本地已有的 xmake 或 xrepo 可执行文件。
- 不得创建或保留本地 xmake 状态与缓存，包括仓库内的 `.xmake`、`build` 目录以及用户级 `.xmake` 目录。
- C++ 构建验证必须使用现有的 GitHub Actions 工作流 `.github/workflows/build.yml`。
- 本地已经安装 `git` 和 GitHub CLI（`gh`），应使用它们检查提交、工作流、运行日志和构建产物。
- 触发工作流、创建提交或推送仍须得到当前任务的明确授权；获得授权后，应使用 `gh` 和仓库现有远程地址，不得搭建本地 xmake 工具链。
- 允许执行不会安装 C++ 构建工具链、不会创建 xmake 缓存的本地只读检查和非 xmake 静态检查。

## 工作区安全

- 工作区存在未提交改动时，必须保留与当前任务无关的用户改动。
- 未经当前任务明确要求，不得删除生成文件或用户拥有的文件。
